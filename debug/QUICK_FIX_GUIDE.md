# 🔍 快速诊断指南

## 您遇到了"未找到图片上传按钮"错误

别担心！请按照以下步骤操作：

### 第1步：检查基本条件 ✅

1. **确认URL** - 请访问 **https://grok.com/imagine**
2. **登录状态** - 确保已登录 X/Twitter 账号
3. **等待加载** - 等待页面完全加载（至少5秒）

### 第2步：查看扩展日志 📋

打开扩展弹窗，查看日志区域的输出，您应该能看到：
```
当前页面: https://grok.com/...
页面共有 X 个文件输入元素
```

**如果看到 "页面共有 0 个文件输入元素"**：
- 说明页面可能需要手动操作才能显示上传区域
- 尝试在页面上点击"Video"或"Imagine"标签
- 刷新页面后重试

**如果看到 "页面共有 1 个或多个文件输入元素"**：
- 扩展已经检测到上传元素
- 查看后续的诊断信息了解为什么无法使用

### 第3步：运行诊断脚本（可选）🔧

如果问题仍未解决，请运行诊断脚本获取更详细信息：

1. 在 grok.com 页面按 **F12** 打开开发者工具
2. 切换到 **Console（控制台）** 标签
3. 访问 GitHub 仓库中的 `debug/page-diagnostic.js`
4. 复制全部内容，粘贴到控制台，按回车运行
5. 将输出结果截图或复制文本

### 第4步：反馈信息 💬

如果以上步骤无法解决问题，请提供：
- 扩展日志截图（显示诊断信息部分）
- 当前页面URL
- 页面截图
- 诊断脚本输出（如果运行了）

## 常见解决方案

### 方案A：页面未正确加载
**现象**：日志显示 "页面共有 0 个文件输入元素"

**解决方法**：
1. 刷新 grok.com 页面
2. 等待页面完全加载（看到上传区域）
3. 再次使用扩展

### 方案B：需要手动进入视频模式
**现象**：在grok.com首页或聊天页面

**解决方法**：
1. 访问 https://grok.com/imagine
2. 如果页面有标签，点击"Video"或"Image-to-Video"
3. 确认看到图片上传区域
4. 使用扩展

### 方案C：元素被隐藏
**现象**：日志显示 "发现 X 个隐藏的文件输入元素"并且显示"尝试使用隐藏的图片输入元素"

**说明**：扩展已自动尝试使用隐藏元素，这是正常的优化行为。如果仍然失败，可能是页面结构变化。
